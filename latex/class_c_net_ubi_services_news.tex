{\footnotesize\ttfamily \mbox{\hyperlink{struct_void}{Void}} C\+Net\+Ubi\+Services\+News\+::\+Get\+Link\+Info (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{struct_integer}{Integer}}}]{Link\+Index,  }\item[{\mbox{\hyperlink{struct_text}{Text}}}]{Type,  }\item[{\mbox{\hyperlink{struct_text}{Text}}}]{Param,  }\item[{\mbox{\hyperlink{struct_text}{Text}}}]{Action\+Name,  }\item[{\mbox{\hyperlink{struct_text}{Text}}}]{Action\+Description }\end{DoxyParamCaption})}

Returns Type, Param, Action\+Name and Action\+Description of the link. Type allows the game to correctly interpret param. The value may be game-\/specific but there are the following predefined types\+: \char`\"{}pstore\char`\"{} means that the param field contains the D\+LC ID related to the news content, \char`\"{}sstore\char`\"{} means that the param field contains an Offer ID related to an offer being available on the secondary store. \char`\"{}weblink\char`\"{} means that the param field should be an U\+RL to a web page. Param defines the parameter value of the action. For a \char`\"{}pstore\char`\"{} type, value must be the D\+LC ID of the 1st party store (platform specific). For example, P\+S3 would look like \char`\"{}\+U\+P0001-\/\+N\+P\+U\+B90687\+\_\+00-\/\+A\+C\+E\+M\+B\+E\+R\+S\+V\+I\+D\+E\+O001\char`\"{}, while on Xbox, it would look like\+: \char`\"{}0x555308796ea1e800\char`\"{}. For a \char`\"{}sstore\char`\"{} type, value must be an Offer ID of the secondary store. This depends on the Secondary Store implementation of the game. For a \char`\"{}weblink\char`\"{} type, value must be a valid H\+T\+TP address. Action\+Name is The name of the action for tracking purposes. Action\+Description is the description of the action the game should display. The game must support default descriptions such as \char`\"{}\+Buy\char`\"{} for a store link. \mbox{\Hypertarget{class_c_net_ubi_services_news_a0979aa7549a5e4ab7e6b8de59fbf03b6}\label{class_c_net_ubi_services_news_a0979aa7549a5e4ab7e6b8de59fbf03b6}}
